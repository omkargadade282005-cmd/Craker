Project Name

Advanced Zip File Password Cracker

Abstract

The Advanced Zip File Password Cracker is a GUI-based desktop application built with Python and PyQt6 that allows users to recover lost or forgotten passwords from encrypted Zip archives. The tool supports two attack modes: Dictionary Attack (using a wordlist file) and Brute Force Attack (using configurable character sets and length ranges). It leverages the pyzipper library to handle modern AES-256 encrypted Zip files – a capability that the standard Python zipfile module lacks. The application uses a multi-threaded architecture (QThread) to ensure the GUI remains responsive during long cracking sessions. Users can start, stop, and monitor the cracking process in real-time through a log window and status indicators.

Features

  1. Dictionary Attack – Load any wordlist file (e.g., rockyou.txt) and iterate through it line-by-line to test passwords.
  2. Brute Force Attack – Generate all possible password combinations from configurable character sets (lowercase, uppercase, digits, symbols) with min/max length controls.
  3. AES-256 Support – Uses pyzipper to crack modern AES-encrypted Zip archives, not just legacy ZipCrypto.
  4. Multi-threaded Engine – Cracking runs on a separate QThread so the GUI never freezes.
  5. Real-time Logging – Live log output showing current attempts, speed, and status.
  6. Start / Stop Controls – Cleanly stop a running attack at any time.
  7. Dark Theme – Modern dark UI via pyqtdarktheme.

Implementation Details

Core Engine (core/worker.py)

The CrackerWorker class inherits from QThread and runs the password-testing loop on a background thread. It opens the zip archive using pyzipper.AESZipFile, then attempts to read the first file in the archive with each candidate password. If the read succeeds without a RuntimeError, the password is correct.

Signals (password_found, stats_updated, progress_updated, error_occurred, task_finished) are used to communicate results back to the main UI thread safely.

User Interface (ui/main_window.py)

The main window is built with PyQt6 widgets arranged in grouped sections: file selection, attack mode configuration (radio buttons toggling between dictionary and brute-force panels), control buttons, a progress bar, and a scrollable log area. All user interactions trigger signals that are wired to the cracker engine.

Utilities (utils/helpers.py)

Helper functions generate the character set from user selections and produce password candidates via Python's itertools.product as a memory-efficient generator.
